# Validation run — full-lifecycle dogfood on `cartkit` (2026-06-17)

A start-to-finish run of Throughline against a fresh demo target, with each persona played by
a **Claude Haiku** subagent, to exercise every part of the framework in action and find what
breaks. The headline result: the independent Reviewer caught a **citation-fraud** defect on code
that was correct and fully green — exactly the "cite-or-don't-ship" guarantee — and the
constitutional retry loop then closed it to PASS.

## The target

`cartkit` (created for this run at `X:/Work/cartkit`): a tiny Python library with one function,
`add_to_cart(cart, item_id, qty, stock)`, a four-rule product spec (`SPEC.md`), and a stdlib
`unittest` suite. The initial commit enforces FR1 (positive qty), FR3 (accumulate), and FR4
(no mutation) but **not FR2** (reject over-stock adds). The three happy-path tests pass, so the
function "looks finished" — the FR2 gap is the spec-completeness bug the framework targets.

- Base commit: `d68e32a`, default branch `master`.
- Slice: `001-enforce-stock-limit` (LOW complexity), spec at `specs/001-enforce-stock-limit/spec.md`.

## What this proves

The central claim held end to end: grounded tests surfaced the FR2 bug that the happy-path
suite missed, and the independent Reviewer — re-reading standards from source — refused to
rubber-stamp a green, correct change because its provenance citation didn't trace to the source
text. That is the difference between "tests pass" and "reviewed against the rules."

## Findings (what dogfooding surfaced) and what was changed

1. **Hooks were Windows-only and depended on Python.** The wiring hardcoded PowerShell, and the
   POSIX `.sh` guards parsed JSON with `python3`. Fixed: per-OS wiring generated by
   `tools/setup-hooks.{ps1,sh,py}` into the gitignored `.claude/settings.local.json` and
   `.codex/hooks.json`; the `.sh` guards now use a JSON parser only if present and a plain
   `grep`/`sed` fallback otherwise (the Bash guard scans the raw payload in fallback mode — it can
   only over-block). Net: the hooks run on Windows/macOS/Linux **with no Python**. Both paths were
   tested (PowerShell guards, and POSIX guards with no interpreter on `PATH`).

2. **A Decision Record cited a non-existent standard.** The catch above was real and valuable, but
   it was made easy by loose guidance. `dev.implement.md` now requires the standard clause to be a
   **real rule id copied from source** (or an explicit "none — convention-only"), and states that
   the Reviewer FAILs citation fraud — so implementers aim for a real clause up front.

3. **A subagent edited an out-of-scope file.** One Haiku persona "helpfully" filled in the clone
   URL in the repo's `README.md`, outside the slice and outside its declared write surface. Harmless
   here and reverted. Honest limitation: the write hooks protect only `standards/`/`exemplars/`, not
   arbitrary repo files — scope discipline elsewhere rests on persona instructions + review + the
   human merge, which is consistent with what STATUS.md already says is "instructed, not enforced."

4. **`wiki/log.md` auto-logging depends on the hooks being wired.** In this run the personas were
   stand-in subagents without the target session's hooks active, so the PostToolUse logger didn't
   fire and the personas didn't hand-append. With hooks wired (now cross-platform) the logger runs;
   the manual-append instruction in the runbooks remains "instructed-only" and easy for a model to skip.
